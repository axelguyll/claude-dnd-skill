# Domain docs

Layout: **single-context**.

- Glossary: `CONTEXT.md` at repo root — canonical vocabulary for the DM skill's domain
  (beats, spines, arcs, tones, state vocabulary). Maintained by `domain-modeling`.
- Decisions: `docs/adr/NNNN-<slug>.md` — hard-to-reverse architectural/domain decisions.
- Companion maps: `docs/ARCHITECTURE.md` (subsystem/data-flow map),
  `Vault\projects\claude-dnd-skill-compass.md` (fragile links, open questions).
